This page covers the technical aspects of this website — its overall structure, style explanations, graphic design aspects, and further discussions.

Overall Structure

The website consists of 7 key items under the websystems/ directory:

Filename File type Purpose
home HTML Introduction to me and my website.
past HTML Past experiences with computing.
future HTML Future aspirations.
comments HTML The technical aspects of the website.
websystems CSS Provide styling for all HTML files.
images Directory Store the favicon and diagrams/pictures.
fonts Directory Store TrueType fonts that are used as the primary font for all pages.

Each page is mostly comprised of boilerplate HTML to define the header and footer and their contents.

The header is comprised of the website's title and a series of anchors (styled as buttons) to navigate between the pages, while the footer includes some additional information as well as a sitemap.

The reason why each page has both a navbar and a sitemap is to provide navigation controls on both the top and bottom of the page, minimising the need to scroll in some cases.

Internal Structure

Wrappers

  1. div#main-wrapper: Serves as a container for the entire page.
  2. div#content-wrapper: Serves as a container for the header, main content and footer.

A hr element is placed after #content-wrapper to mark the end of the header and the start of the main content.

Header

  1. h1: The title of the page - WebSys Assignment.
  2. nav and nav a.btn: A navbar with links to all the pages.

Main Content

The main content of the page, such as p and heading elements are stored in a single container, main#content. The main element is a semantic HTML element that indicates where the main content of the page is stored. Using it was appropriate for this case.

Footer

  1. div.footer-content: Contains two divs displayed in a flexbox row layout, which include the course number and sitemap.
  2. p: Holds the copyright disclaimer.

Like the space between the header and the main content, the children of the footer are also divided by a hr element.

CSS Classes

This section will provide a brief outline of the CSS classes that were used.

  • btn: Styles elements that perform an action, e.g. buttons.
  • bg-primary, bg-secondary, etc: Primarily used to demonstrate the colour scheme within <span> elements.
  • border-codelike: Apply a style like that of a <code> element.
  • nowrap: Prevent an element's text from wrapping in a way that breaks words. Used in table cells to ensure code examples aren't broken.
  • active-page: Applied to the button which is used to navigate to the page the user is currently viewing.
  • sitemap: Style the sitemap element.

CSS Styles

This section covers some of the more complex style properties, beyond self explanatory ones like color. Each style is first explained in detail, followed by a brief look at how specific values were applied in the website’s design.

Name Explanation
box-sizing The box-sizing property comes with two options, content-box (the default) and border-box, which was used for this page.

Content box sizing interprets the padding and border of an element to add on to its defined width, whereas border box sizing intrinsically includes the padding and border of an element to fit within its defined width. The border-box sizing property is far more intuitive than its counterpart which is why it was used.
display The display property outlines how an element should be rendered in the DOM. For this website, the flex value was primarily used. The inline value was also used a single time.

The flexbox layout allows elements to be arranged by columns or rows, and is very flexible when it comes to positioning elements and defining their proportions. This page uses a flexbox display within the header and the footer, as well as the content wrapper.

The inline layout makes elements behave like text. This was used in the footer sitemap to force all list items containing the anchor elements into a single line.
justify-content justify-content controls how the space in between flex items is distributed along the primary axis of a flex container. The site used various different values for this style.
  • space-between: Evenly distributes the space between items, while ensuring that the first and last item are aligned with the starting and finishing edges. It is used in .footer-content to position the course number section opposite the sitemap.
  • space-evenly: Ensures the space between and around the items is equal, providing the most uniform distribution out of all the values for justify-content. It is used in header to distribute the heading and navbar.
  • center: Pushes all items flushly together into the center of the flex container. It is utilised in the nav to center the nav once the flex-direction of the parent container switches to column.
It is important to note that this property has a "sibling", which is align-content. The difference between them is that justify-content affects distribution in the direction of the main axis (set to row by default), while align-content affects item distribution in the direction of the cross-axis (meaning single row/single column flex containers are unaffected by align-content).
padding Padding is the distance between an element's content and its border. Almost every element, including this very paragraph, has padding.
border The border is what wraps around a padded element. It essentially sits in between the padding and margin of an element. This site frequently uses borders, like those around buttons, <code> elements, tables, and the header/footer.
margin An element's margin defines how far it is spaced from nearby elements. When considering the padding-border-margin trifecta, the margin is the outmost layer.

CSS Combinators

This section outlines some of the CSS combinators that were used when styling content. A combinator is used to combine selectors in a way that allows elements to be selected based on their location relative to other elements.

Name Explanation Example
Selector list A comma separated list of selectors that, allowing you to select multiple elements to apply the same styles to. This combination method was used frequently throughout the website. h1, h2, h3
Descendant combinator The descendant combinator (denoted by a space) combines two selectors in a way that requires the first selector to be a parent (direct or indirect) of the second selector. This is another combinator that was used often throughout the CSS, in places such as <table> elements on <li> elements nested within different containers. .sitemap li
Child combinator The child combinator (>) is used to select an element that is an immediate descendant of the first element. It was used in the website to style all elements inside the table header row element (of specificity table thead tr). #id div > h1
Subsequent-sibling combinator The subsequent-sibling combinator (~) allows you to select all elements that are preceded by a certain element. The website utilises this combinator to apply margins to headers if they are preceded by content, using these selectors: #content * ~ h1, * ~ h2, ..., which selects header elements that are preceded by any element (*). If a header is not preceded by anything, it means that is it at the beginning of the #content div, and should not have any top margins. h1 ~ p
Next-sibling combinator The next-sibling combinator (+) selects all the elements that follow a certain element. It was used in the website's footer to italicize the first paragraph that followed a <h3> (the course identification). h3 + p

Graphic Design Aspects

Colouring

Due to the lack of variation in website content, with the only diversity coming from buttons and text elements that carry more importance than just regular paragraphs, an extremely simple colour palette was formulated. It consists of three consecutively darker shades of white for primary colouring, as well as additional shade for alternate table rows, and three consecutively darker orange accent shades for elements such as headings (relating to primary discussions), links, focused elements, and selected elements.

Typefaces

This website uses three typefaces of Inter (bold, normal, and italic), an open source free sans-serif font that emphasises high readability and aesthetics. If the user's browser does not support Inter, two fallback fonts were provided to the font-family property: Arial, and sans-serif (sans-serif in this context instructs CSS to find the operating system's default sans-serif font).

Layout

Various layout features have been implemented to improve the ergonomics of the website.

  • Max width of #main-content: The maximum width of the main content container has been set to 960px. Without this restraint, text would expand across the entire screen, making it harder to read.
  • Responsive HTML: Through the use of media queries, the website can adapt to different screen sizes, such as the alternation between column and row flexbox layouts.
  • Padding: Almost every element has some amount of padding to make it distinguishable. This approach prevents a crammed website design.
  • Margins: Margins are used to centre the main content and add spacing between headings to reflect their hierarchy and importance.
  • Interactivity: Navbar buttons change colour depending on their state (using CSS pseudo-selectors like active and focus). Furthermore, the navbar buttons also indicate which page is active through bolded text.
  • Use of the <code> element: Placing content that refers to HTML, CSS, or computer science related concepts in code tags helps support the key ideas of the comment page, while also making it easier for the reader to understand the primary concepts that are being discussed.

Accessibility and Ergonomics

The Web Accessibility Initiative (WAI) was adhered to when developing the website, to accomodate for a wide range of end-users. Thorough consideration was also given to various other design aspects to improve ergonomics.

Consideration of screen readers

All images used in the website have a defined alt attribute, which describes the content of the image. This is a vital design consideration for screen readers (often used by visually impaired users), as the alt property takes the place of the image.

Compatibility with smaller devices

To ensure the website could be viewed on smaller devices, Firefox's Responsive Design Mode feature was utilised. The website was tested on a variety of dimensions to ensure it was legible and still looked good.

Font sizing, spacing and selection

The font size was reduced from 16 pixels to 15.5 pixels, which suited the design of the website better. Line height was increased to 1.6rem to improve readability. The use of the rem unit is vital in this case, as it allows the line spacing to scale with the root font size (what rem) refers to.

Semantic Elements

While most HTML elements carry little to no semantic value, such as the <div> element, some elements such as <h1...h6> and <aside> carry more meaning. One such example is to compare the <b> and <strong> elements. The former simply denotes text that is stylised as bold. The latter denotes text that is important to the context it is placed in. Semantic elements such as these gives the website a meaningful structure while also helping screen readers distinguish what part of the page the user is in (a benefit that relates more to semantic elements like <main>).

Anchors

All anchors have the target attribute set to _blank, which makes the anchor's url open in a new tab. It can be annoying when interacting with an anchor overrides the current tab's webpage, so this feature improves the website's ergonomics and useability.

No style appearance

When disabling styles, the website renders well enough that the content can be read and distinguished. One issue was images being too large, so, instead of changing their resolution, the width attribute of the <img> element was set to an integer between 300-600, depending on the image's aspect ratio, to prevent it from exceeding the size of the page. The website's ability to be read and understood without CSS accomodates for some screen readers or viewers who prefer CSS off.
